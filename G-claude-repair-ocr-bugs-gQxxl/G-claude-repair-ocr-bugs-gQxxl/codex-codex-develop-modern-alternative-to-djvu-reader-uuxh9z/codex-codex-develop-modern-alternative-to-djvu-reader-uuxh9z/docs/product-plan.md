# Продуктовый план: NovaReader 2.0.0

## 1) Цель релиза 2.0.0

Выпустить **крупное обновление качества и производительности**, в котором:
- все ключевые функции чтения, аннотирования, OCR и экспорта работают стабильно;
- устранены подвисания интерфейса и долгие блокировки при работе с инструментами;
- PDF-сценарии расширены до полноценного уровня редактирования и конвертации.

Главный критерий успеха: пользователь может открыть документ, редактировать/искать/распознавать/экспортировать без критических ошибок и деградации UX.

---

## 2) Принципы разработки 2.0.0

1. **Stability First** — сначала надежность и предсказуемость, затем расширение функционала.
2. **Performance by default** — тяжелые операции не должны блокировать UI.
3. **Measurable quality** — у каждой функции есть SLA/метрики и acceptance criteria.
4. **Progressive delivery** — релиз через этапы с feature flags и канареечным тестированием.

---

## 3) Ключевые направления 2.0.0

## A. Базовая стабильность и отказоустойчивость
- Единая система обработки ошибок (runtime + UI).
- Автовосстановление состояния документа/сессии после ошибок.
- Журнал событий (debug/telemetry) для диагностики падений.
- Сценарии graceful degradation (если OCR/конвертер/модуль недоступен).

**Definition of Done:**
- критические пользовательские сценарии без фатальных ошибок;
- не менее 99% успешных открытий документов на тестовом наборе.

## B. Производительность ядра и инструментов
- Профилирование рендера страниц, аннотаций, поиска, OCR.
- Перевод тяжелых задач в Web Workers (OCR, индексирование, preprocessing).
- Виртуализация длинных списков/панелей, debounce/throttle UI-событий.
- Оптимизация памяти (ограниченные кэши, LRU, освобождение ресурсов).

**Целевые метрики:**
- TTI (интерактивность) после открытия документа: < 2 сек для типовых PDF;
- переключение страницы: p95 < 120 мс (кэш) / < 350 мс (без кэша);
- не более 1 долгого main-thread task > 50 мс в типовом сценарии аннотаций.

## C. OCR 2.0 (качество и скорость)
- Новый OCR pipeline: pre-processing (deskew/denoise/binarization/contrast),
  language hints, confidence scoring, post-correction.
- Deskew/выравнивание текста и строк как отдельный этап с адаптивными порогами.
- Batch OCR по документу с очередью задач и отменой.
- Интеграция OCR-индекса в поиск и экспорт.

**Целевые метрики:**
- CER/WER улучшение минимум на 25% на внутреннем датасете сканов;
- время OCR страницы A4 300dpi: p95 < 2.5 сек на референсной машине;
- для страниц с перекосом > 3°: заметное снижение ошибок распознавания.

## D. PDF Pro: редактирование, исправление, экспорт
- Редактор PDF-слоя (текст/блоки/позиционирование) с историей undo/redo.
- Исправление OCR-слоя и синхронизация правок с документом.
- Экспорт PDF → DOCX с сохранением структуры:
  - абзацы/стили,
  - таблицы,
  - изображения,
  - базовое форматирование.
- Импорт обратно DOCX-правок (опционально в рамках 2.0.x).

**Целевые метрики:**
- безошибочный экспорт на smoke-наборе (не менее 95% тест-кейсов без критической потери структуры);
- таблицы и изображения корректно переносятся в >= 90% кейсов целевого набора.

## E. UX и функциональная целостность
- Переработка сценариев “открыть → читать → редактировать → экспортировать”.
- Устранение конфликтов hotkeys и гонок состояний инструментов.
- Проверка, что все панели и настройки реально применяются и сохраняются.
- Улучшение адаптивности под экраны меньше 16:9.

**Целевые метрики:**
- 0 blocker/critical багов перед GA;
- SUS/внутренний UX score >= 80.

## 4) Архитектура 2.0.0 (целевые изменения)

- Выделить модули:
  - `render-core`
  - `annotation-engine`
  - `ocr-engine`
  - `pdf-edit-engine`
  - `export-conversion`
- Добавить task scheduler + worker pool.
- Ввести typed contracts между модулями (JSDoc/TS migration-ready).
- Подготовить compatibility layer для 1.x workspace.

## 5) План поставки (этапы)

## Milestone M1 — Stabilization Foundation (2–3 недели)
- bugfix wave, единая обработка ошибок, baseline telemetry.
- performance baseline + профилирование top-20 сценариев.

**Выход:** beta-internal #1.

## Milestone M2 — Performance & UX Core (2–3 недели)
- workerization OCR/index/search; оптимизация рендера и аннотаций.
- устранение UI-freeze, стабилизация layout/settings.

**Выход:** beta-internal #2.

## Milestone M3 — OCR 2.0 (3 недели)
- deskew/preprocess pipeline, confidence, batch OCR queue.
- regression suite OCR quality + speed.

**Выход:** RC-ocr.

## Milestone M4 — PDF Pro (4 недели)
- PDF editing core, undo/redo, PDF→DOCX converter v1.
- корректировка структуры таблиц/изображений.

**Выход:** RC-feature-complete.

## Milestone M5 — Hardening & GA (2 недели)
- e2e, soak tests, crash-free pass, документация, финальная полировка.

**Выход:** **NovaReader 2.0.0 GA**.

## 6) Качество и тестирование

- Unit tests: ядро рендера/состояний/утилит.
- Integration tests: OCR pipeline, PDF edit/save, converter.
- E2E tests: пользовательские сценарии (Playwright).
- Performance tests: CI профили + budget gates.
- Corpus tests:
  - сканы плохого качества,
  - большие PDF,
  - документы с таблицами/графикой/многоязычностью.

**Quality Gate 2.0.0:**
- 0 blocker/critical open defects;
- crash-free sessions >= 99.5% на канареечном прогоне;
- все p95/perf budgets в зеленой зоне.

---

## 7) Риски и mitigation

- Риск: сложность “редактирование PDF как в word”.
  - Mitigation: staged scope (v1 text-block editing + OCR correction), feature flags.
- Риск: качество PDF→DOCX на сложной верстке.
  - Mitigation: rule-based + fallback model, corpus-driven QA, ручной review для edge cases.
- Риск: regression по производительности.
  - Mitigation: perf budgets в CI, обязательный profiling report на каждый крупный PR.

---

## 8) Definition of Complete для 2.0.0

Релиз 2.0.0 считается завершенным, когда:
1. Пользовательский поток “открыть → обработать → исправить → экспортировать” проходит без критических ошибок.
2. OCR и PDF-инструменты достигают целевых метрик качества/скорости.
3. Интерфейс остается отзывчивым под реальной нагрузкой.
4. Все основные функции приложения подтверждены автотестами и regression-набором.
