# NovaReader — План редизайна интерфейса

## Текущие проблемы

1. **Мелко** — базовый шрифт 13px, кнопки `.btn-xs` имеют padding 3px 6px и шрифт 9.36px (0.72rem), toolbar кнопки ещё мельче (формула `0.62rem + 0.12rem * 1` = ~9.6px). Нечитаемо на больших мониторах.
2. **Неинформативно** — кнопки используют Unicode-символы (↶, ◂, ▸, ⊡) вместо иконок, нет описательных лейблов, статус-строки слишком мелкие (0.72rem = 9.36px).
3. **Много лишних кнопок** — 91+ кнопка на экране одновременно. Только в text-tools 28+ кнопок в двух рядах. Sidebar содержит 47+ кнопок. Все видны постоянно.
4. **Неудобные пропорции** — sidebar 240px, top toolbar 36px, bottom toolbar 80px, text panel 110px. Canvas (область просмотра) получает мало места. Bottom toolbar перегружен.

## Фаза 2: Реорганизация тулбаров — progressive disclosure

### 2.1 Верхний тулбар (основные действия)
Оставить только самые частые действия (10-12 кнопок):
```
[← →] [◂ стр. ▸] [pageInput] | [zoom- % zoom+ ширина страница] | [🌓 ⛶ ⚙]
```
- Убрать: `↻ Повернуть` → в dropdown-меню "Вид"
- Убрать: `⌨ Горячие клавиши` → в настройки
- Убрать: toggle-кнопки панелей (`☰ ▤ ¶ ⌕ ✎`) → перенести в меню "Вид" (dropdown)

### 2.2 Нижний тулбар → Контекстная панель инструментов
Разделить на **вкладки** вместо одной длинной строки:
```
[🔍 Поиск] [✎ Аннотации] [📄 Текст/OCR] [📎 Инструменты]
```

Каждая вкладка показывает только свою группу:
- **Поиск**: searchInput, scope, найти, ↑↓, статус
- **Аннотации**: toggle, drawTool, цвет, размер, undo, clear, экспорт
- **Текст/OCR**: refreshText, copy, .txt, .docx, OCR, область, качество
- **Инструменты**: формы, блоки, изображения, водяной знак, штамп, подпись, merge/split

### 2.3 Выпадающие меню (dropdown)
Кнопки экспорта/импорта сгруппировать в dropdown-меню:
- «↗ Экспорт» → dropdown: PNG, SVG, PDF, JSON стр., JSON док.
- «↙ Импорт» → dropdown: JSON стр., JSON док.
- Аналогично для текстовых экспортов

**Файлы:** `index.html`, `styles.css`, `app.js` (обработчики вкладок и dropdown)

## Фаза 3: Редизайн сайдбара

### 3.1 Компактная навигация
- Убрать секции по умолчанию скрытые (advanced sections) из основного вида
- Оставить видимыми: **Недавние**, **Закладки**, **Оглавление**, **Превью**
- Прогресс, история, комментарии, заметки — доступны через **Tab bar** или **аккордеон** с ленивой загрузкой

### 3.2 Tab bar сайдбара
```
[📑 Навиг.] [📝 Заметки] [📊 Аналит.] [⚙ Настр.]
```

- **Навигация**: Недавние, Закладки, Оглавление, Превью
- **Заметки**: Заметки, Workspace
- **Аналитика**: Прогресс, Статистика, Переходы, Результаты поиска, История поиска
- **Настройки**: Настройки сайдбара, Cloud & Collab, Диагностика

### 3.3 Кнопки в секциях
Заменить множество мелких кнопок на:
- **Контекстное меню** (правый клик или "⋮" кнопка) для Import/Export
- Одна кнопка "Действия" → dropdown список

Пример для секции "Закладки":
```
Было:  [+ Добавить] [Очистить] [↗ .json] [↙ .json] [Фильтр] [✕]
Стало: [+ Добавить] [Фильтр _____ ✕] [⋮ меню → Экспорт JSON / Импорт JSON / Очистить]
```

### 3.4 Ширина и пропорции

- Увеличить padding секций: 8px → 12px

## Фаза 5: Визуальная полировка

### 5.1 Кнопки с иконками
Каждая основная кнопка содержит SVG-иконку + текст:
```html
<button id="searchBtn" class="btn-tool">
  <svg>...</svg> <span>Найти</span>
</button>
```
В компактном режиме текст скрывается, остаётся иконка + tooltip.

### 5.2 Статус-бар
Добавить нижний статус-бар (как в VS Code):
```
[Страница 5/120] [Zoom 100%] [OCR: готов] [Размер: 2.4 МБ] [Время чтения: 12 мин]
```
Вместо разбросанных по интерфейсу status-line элементов.

### 5.3 Цветовая схема
- Активные вкладки: чёткое выделение цветом `--accent`
- Disabled кнопки: `opacity: 0.4` + `cursor: not-allowed`
- Badge-счётчики на вкладках (кол-во закладок, результатов поиска)

### 5.4 Анимации
- Плавное переключение вкладок (opacity transition 150ms)
- Sidebar collapse/expand: width transition 200ms
- Dropdown появление: transform + opacity 150ms

**Файлы:** `styles.css`, `index.html`

## Фаза 6: Settings Modal — упрощение

### 6.1 Текущая проблема
Modal настроек содержит слишком много технических параметров (px размеры панелей, annotation canvas scale). Обычный пользователь не должен это трогать.

### 6.2 Решение: вкладки в модале
```
[Общие] [Внешний вид] [OCR] [Клавиши] [Продвинутые]
```

- **Общие**: Тема, Язык, показ/скрытие панелей
- **Внешний вид**: Размер интерфейса (ползунок S/M/L), видимость секций
- **OCR**: Язык OCR, режим точности, фоновое OCR, хранилище
- **Клавиши**: Горячие клавиши
- **Продвинутые**: Все px-размеры, annotation canvas, диагностика, self-check

### 6.3 Preset-размеры вместо px
Вместо 8 отдельных полей ввода в пикселях — один ползунок:
```
Масштаб интерфейса: [S ───●─── L]
```
Который пропорционально меняет все CSS-переменные.

**Файлы:** `index.html`, `styles.css`, `app.js`
