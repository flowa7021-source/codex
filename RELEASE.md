# RELEASE.md

## Release target
- Product: OfflineDocStudio
- Release: **1.0.0**

## Artifacts
1. ZIP package (CPack):
   - `OfflineDocStudio-<version>-<platform>.zip`
2. Windows installer (Inno Setup):
   - `dist/OfflineDocStudio-Setup-<version>.exe`
3. Windows portable bundle:
   - `dist/OfflineDocStudio-<version>-portable`
   - `dist/OfflineDocStudio-<version>-portable.zip`
4. Artifact manifest:
   - `dist/RELEASE_ARTIFACTS.txt`

## Release commands (Linux/macOS)
```bash
cmake -S . -B build
cmake --build build -j
ctest --test-dir build --output-on-failure
cpack --config build/CPackConfig.cmake
```

## Release commands (Windows)
```powershell
./scripts/build_windows_installer.ps1
```

## Quick installer build (cross-env)
```bash
./scripts/build_installer_now.sh
# strict mode: fail if Inno Setup path is not used
./scripts/build_installer_now.sh --require-native
```

## Verification checklist
- `scripts/verify_release.py` проходит успешно (версии синхронизированы).
- `scripts/verify_installer_payload.py` проходит успешно (payload инсталлятора валиден).
- `scripts/verify_windows_release.ps1` проходит успешно (Windows release prerequisites валидны).
- Build and tests pass.
- `--doctor` reports healthy runtime layout after `--init-layout`.
- Interactive mode runs and exits cleanly.
- ZIP package is generated by CPack.
- `dist/RELEASE_ARTIFACTS.txt` contains hashes and sizes for produced artifacts.
- Inno Setup script is up-to-date with current release version.
- `scripts/verify_installer_ready.py` confirms installer file + PE format + manifest hash consistency.
- `dist/INSTALLER_METADATA.json` is generated for machine-readable installer release metadata.


## CI artifacts
- Windows job публикует пакет `OfflineDocStudio-windows-release-<Configuration>` (installer + portable + manifest), который можно скачать и сразу протестировать без локальной Windows-сборки.
- Для ручной релизной проверки используется `workflow_dispatch` (настройка конфигурации и включение/отключение шага сборки инсталлятора).


## Local install smoke check (Linux)
- `./scripts/install_local.sh --prefix /tmp/ods-install`
- `/tmp/ods-install/bin/offline_doc_studio --version`

- Автоматизированная выгрузка Windows installer: `scripts/fetch_windows_release.sh --repo <owner/repo> --configuration Release --build-installer true` (через `gh` или через `GITHUB_TOKEN` API fallback).

- Linux fallback installer build: `./scripts/build_windows_stub_installer.sh` -> `dist/OfflineDocStudio-Setup-<version>.exe` (self-contained installer, устанавливающий payload в `%LOCALAPPDATA%\OfflineDocStudio` или в путь из параметра запуска `/D=<путь>`).

- One-shot release+installer: `./scripts/release_with_installer.sh`.

- Alternative installer format (NSIS): `./scripts/build_installer_now.sh --format nsis`.
