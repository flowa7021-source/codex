# Техническое задание‑промт для разработки офлайн‑приложения‑аналога PDFGear

## Цель
Создать полностью офлайн‑приложение для Windows 10–11 (ридер/аннотатор), ориентированное на UX PDFGear (лента + Home‑центр + быстрые инструменты), с расширенной поддержкой форматов **PDF / DjVu / CBZ / EPUB** и portable‑подходом.

---

## 1) UX‑ориентиры и продуктовая модель

### 1.1 Home как центр управления
- Стартовая страница Home/Library:
  - Open File
  - Recent
  - Pinned
  - Библиотека/полка (поиск, фильтры, коллекции)
  - Быстрые действия (импорт папки, очистка истории, настройки)
  - Шпаргалка hotkeys
- «Нулевая фрустрация первого клика»: пользователь сразу понимает, что делать.

### 1.2 Document Page + Ribbon
- Лента с вкладками, но **Home‑вкладка содержит все инструменты** (дополнительные вкладки допустимы как дубль).
- Боковая панель: миниатюры / закладки / комментарии.
- Режимы работы: Reading Mode (минимум UI) и Edit/Annotate Mode.

### 1.3 Home‑инструменты (обязательный минимум)
- Режимы просмотра и масштаб
- Поиск
- OCR
- Screenshot / Snapshot
- Copy text / Copy selection as image
- Аннотации
- Страницы (delete/rotate/crop)
- Печать

---

## 2) Обязательные форматы и офлайн‑ограничения

### 2.1 Форматы
- PDF
- DjVu (`.djvu`, `.djv`)
- CBZ
- EPUB

### 2.2 Offline‑first (жёстко)
Запрещены любые сетевые обращения в рантайме:
- check updates
- телеметрия
- облачные AI/OCR
- CDN/внешние web‑ресурсы
- онлайн‑конвертеры

### 2.3 Portable‑принцип
- Не писать настройки в реестр.
- Хранить конфиги рядом с exe (`config`, `hotkeys`, `library`, OCR‑кэш, sidecar‑данные).
- Ассоциации файлов — только через установщик или отдельную helper‑утилиту по явному действию пользователя.

---

## 3) Функциональные требования (контракт поведения)

### 3.1 Режимы чтения/навигации
- Single page
- Continuous scroll (с опцией минимальных интервалов/без рамок)
- Pages Overview (grid)
- Two pages (facing)
- Book
- Booklet
- Fullscreen
- Slide Show

### 3.2 Масштаб и fit‑режимы
- Fit Width / Fit Page / Actual Size / Custom
- Fit Height
- Fit по меньшей стороне
- Шаги zoom: 10/25/33/50/66/75/100/125/150/200 + ручной ввод
- `Ctrl + wheel` = zoom
- Double click = настраиваемое zoom‑действие
- `Space (hold)` = временный Hand tool
- `Middle-click-drag` = pan

### 3.3 Поиск
- Единый поиск по документу + панель результатов + подсветка.
- PDF/EPUB: нативный текстовый поиск.
- DjVu/CBZ/сканы: OCR‑индексация и поиск по OCR.

### 3.4 OCR (обязательно)
- OCR выделенной области
- OCR текущей страницы
- OCR всего документа (фоновой задачей с прогрессом)
- Выбор языка (минимум RU)
- Результат: Copy text + Save TXT
- Make searchable:
  - sidecar‑индекс (без изменения исходника)
  - либо встраивание text layer в PDF при сохранении
- Кэш OCR по хешу содержимого документа.

### 3.5 Screenshot / Copy as image (обязательно)
- Выделение прямоугольником по странице.
- Действия: Copy PNG в буфер, Save PNG, pin thumbnail.
- Дефолтный хоткей допускается `Ctrl+Alt+X`, но hotkeys обязаны быть редактируемыми.

### 3.6 Аннотации
- Note/sticky
- Highlight / underline / strikethrough
- Ink + eraser
- Textbox / free text
- Stamps
- Панель аннотаций (список, фильтры, переход)
- Hide/Show annotations (toggle)
- Lock/Freeze:
  - lock annotation (свойства/перемещение/удаление)
  - lock contents (содержимое)
  - flatten (неизменяемая визуальная фиксация)

### 3.7 Операции со страницами
- Delete pages
- Rotate 90/180/270
- Crop (auto + manual)
- Для CBZ: операции выполняются как трансформации изображений + пересборка архива.

### 3.8 Печать
- Печать диапазона/выбранных страниц
- Печать с аннотациями / без аннотаций

### 3.9 Табы, сессии, CLI, drag&drop
- Многовкладочный режим
- Восстановление последней сессии
- Drag&drop файлов в окно
- CLI: открыть файл, перейти к странице, выполнить поиск

### 3.10 Надёжность
- Обработка частично повреждённых файлов
- Дружелюбные ошибки + попытка восстановить/упрощённый режим
- Логи + crash dump

## 4) Архитектура (C++/native)

### 4.1 Слои
- UI layer (Ribbon/команды/хоткеи/вкладки)
- Core (Document abstraction, render pipeline, selection, search API)
- Format plugins (PDF, DjVu, CBZ, EPUB)
- Annotation subsystem
- OCR subsystem
- Storage subsystem (portable)
- Printing/export subsystem
- Logging/crash dump subsystem

### 4.2 UI и DPI
- Нативный Ribbon (Windows Ribbon Framework).
- Per‑Monitor DPI awareness (предпочтительно PMv2).
- Корректная обработка DPI‑смены между мониторами.

### 4.3 Рендеринг и производительность
- Progressive render (быстрый черновой → уточнение)
- Виртуализация continuous scroll
- Кэш страниц с жёстким бюджетом памяти и освобождением при закрытии
- Безопасная диспетчеризация вызовов к не‑thread‑safe движкам

### 4.4 Кандидаты движков
- PDF: PDFium / MuPDF / Poppler (с фиксацией лицензий)
- Структурные PDF‑операции: qpdf
- DjVu: DjVuLibre
- EPUB: WebView2 (с жёсткой блокировкой внешних URL) или Readium SDK
- OCR: Tesseract

## 6) Критерии приёмки (Definition of Done)

Релиз принимается только если:
- Все обязательные функции доступны из Home‑вкладки.
- Hotkeys переназначаемы и экспортируемы/импортируемы.
- OCR работает (RU), включая поиск по OCR‑индексу.
- Screenshot hotkey и copy selection as image работают стабильно.
- Аннотации: полный минимальный набор + hide/show + manager + lock/freeze.
- Delete/rotate/crop страниц работают корректно.
- Печать диапазонов (с/без аннотаций) работает.
- Нет лагов/зависаний на больших документах (500+ страниц), нет утечек памяти.
- Приложение не инициирует сетевые запросы.
- Корректная работа на Windows 10–11 и HiDPI.
- Есть логи и crash dumps.

## 8) Готовый роль‑промт для программиста

Ты — ведущий C++ разработчик под Windows 10–11 для высокопроизводительных нативных desktop‑приложений (Win32/COM/Direct2D/DirectWrite). Твоя задача — спроектировать и реализовать офлайн‑приложение‑ридер/аннотатор в стиле PDFGear (Ribbon + Home‑центр команд), поддерживающее PDF/DjVu/CBZ/EPUB, без сетевых запросов и автообновлений, с portable‑хранением настроек рядом с exe.

Обязательства:
1) Реализовать все функции из раздела «Функциональные требования».
2) Поддержать производительность: первый экран <1с, операции навигации целево <200мс.
3) Обеспечить стабильность, логи, crash‑dumps, обработку повреждённых файлов.
4) Выполнить spike выбора движков и зафиксировать EDR.
5) Довести решение до DoD.

Дополнительно (без нарушения минимализма): session restore, command palette, history back/forward, night/invert режим.
